Virtual Memory

Today, OS and applications occupy space in excess of 4GB RAM! With the relatively substantial cost of main memory (system RAM) and most systems being limited to 2GB or 4GB of RAM how does it all fit in a comparatively small main memor? Well, actually, it does not. A virtual effect is however possible by virtue of something known as "Virtual Memory".
Virtual Memory is a technique of executing program instructions that may not fit entirely in system memory. This is done by calling instructions as and when the application requires it. Virtual Memory is implemented by using secondary storage to augment the main memory. Data is transferred from secondary to main storage as and when necessary and the data modified is written back to the secondary storage according to a predetermined algoritm.
Virtual Memory, thus enhances the CPU utilization and throughput by executing as many programs as possible, almost simultaneously.

  • Map some of the program's address space to the disk;

  • When we need it, we bring it into memory.

Supplementing physical memory (RAM) with secondary storage is known as virtual memory.
