The $PATH variable

When you enter an external command in the shell command line interface, the shell must seach the system to find the program. The shell uses an environment variable called PATH to find commands:

echo $PATH

/sbin:/bin:/usr/sbin:/usr/bin

The PATH environment variable is set to look for commands only in a handful of directories, separated by colons. To add a new directory to the PATH variable:

PATH=$PATH:/home/gidna/bin

There must be no spaces between the = sign.

Il file .bashrc, presente nella home directory (/root per l'user root), permette di memorizzare variabili globali e alias:

PATH=$PATH:/home/gidna/bin

ht="/srv/www/htdocs"

Aliases in Linux are environment variables that behave in the same way they sound — as another name for a command, sono cioè nomi alternativi. You can define aliases on-the-fly at the command line, but those are good for only one session. It's preferable to have these available to you all the time.
