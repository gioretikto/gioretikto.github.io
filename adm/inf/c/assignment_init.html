Declaration, Assigment and initialization

Declaring and Defining variables

A variable declaration, is a program statement that specifies the name of a variable of a given type. For example

int age;

This declares a variable with the name age that can store integers. The type of data that can be stored in the variable age is specifies by the keywork int.

In C a variable declaration is also a definition. A variable definition associates a piece of the computer's memory with the variable name, in order to store data. That space is associated with a unique address

You use the term declaration when you introduce a name into your program, with information on what the name will be used for. The term definition refers to the allotment of computer memory to the name. In the case of variables, you can declare and define in a single statement, as in the preceding line. The reason for this apparently pedantic differentiation between a declaration and a definition is that you will meet statements that are declarations but not definitions.

More than one variable of the same type can also be declared in one statement. In this case, the names of the variables are written after the keyword separated by commas. For example, to declare three variable of type int

int a,b,c;

Initialization and Assignment

Two similar but logically distinct, operations:

  • Initialization (giving a variable its initial value)

  • Assignment (giving a variable a new value)

These operations are so similar that C allows us to use the same notation (the = ) for both:

int y = 8; // initialize y with 8
x = 9;    // assign 9 to x

Variable scope

The scope of a variable is the area of the program where that variable is valid, i.e. the parts of the program that have acess to that variable. This is determined by the location of the declaration. There are four types of variables scopes: block, function, file and program scope.

Block or function Scope

Most scopes in C are delimited by curly braces.

A statement block is a group of statements enclosed within an opening and closing curly brackets: {}. If you declare a variable within a statement block, when the program exits the block, the variable will be destroyed from memory. This type of variable, is said to have BLOCK SCOPE, and is also known as a LOCAL VARIABLE. Your program may contain a nested blocks. If you declare and initialize a variable x in outer block and then declare initialize the same variable called x inside the inner block, they will occupy up different memory slots and are considered as different variables.

Program Scope

If you want to avoid passing arguments to your functions you can declare your variables outside any function blocks. Such variables are sometimes known as GLOBAL VARIABLES. Their value can be accessed from anywhere within the ENTIRE program.

File Scope

FILE SCOPE is when a variable is only accessible until the end of the file. A variable with file scope is a global variable declared with the static keyword before the data type, like this: 

static int x = 15
; This makes it impossible for x to be accessible by other files, other than the one it's declared and initialized in. This is useful for wnen you come to write your own header files as it can prevent compilation errors and bugs.

Examples of scopes

void f()
{
	g();		// error: g() isn’t (yet) in scope
}
void g()
{
	f();		// OK: f() is in scope
}
void h()
{
	int x = y;		// error: y isn’t (yet) in scope
	int y = x;		// OK: x is in scope
	g();			// OK: g() is in scope
}
