HTTP (HyperText Transfer Protocol)

HTTP (HyperText Transfer Protocol) is the application layer protocol that form the foundation of the World Wide Web (WWW). HTTP is a request/response protocol, relying on a client-server model. As illustrated in Fig. 1, the client's browser makes a request for Web objects (whether HTML page, an image, a video or a file). The server responses by sending back HTTP data over the same TCP connection that was used for the request. These HTTP responses return the message body, which contains the requested data or content. The response also contain HTTP headers that describe the data.

Some of the fundamental characteristics of the HTTP protocol are:

  • HTTP is stateless protocol, that is each request-response echange is treated independently. Clients and servers are not required to retain a state.

  • HTTP is media independent. Any type of data can be sent by HTTP if bothe the client and the serve know how to handle the data content.

HTTP Headers

HTTP stores information about both requests and responses in HTTP headers, which the browser and the web server read. These headers aren't visibile to users, but they transmit various types of information between client and server, such as what type of data is returned, your deveice's operating system, the type of browser that you're using and the type of content being requested. They also transmit an HTTP status code that reports on the completion of the request:

  • 200 = ok (request completed)

  • 301 = moved permanently (redirection)

  • 404 = not found;

  • 418 = I'm a teapot

  • 451 = unabailable for legal reasons

  • 500 = internal server error

HTTPS

HTTP isn't privacy sensitive. Routers and intermediate devices, such as those controlled by your ISP, and the websites you visit can read and modify all of the information transmitted over HTTP, including the HTTP headers, the originatic and destination IP addresses, and even the response data. This is because HTTP itself isn't encrypted.

HTTPS is not actually an acronym. HTTPS is purely defined as securing HTTP connections by means of the cryptographic protocol Transport Layer Security (TLS).

Transport Layer Security (TLS)

The Transport Layer Security (TLS) is a protocol for providing confidentiality, integrity, and authentication to Internet applicatoins. TLS fits between the application and TCP layers adding encryption and authentication functions above the TCP protocol, thus providing a secure communication service the the application layer (including HTTP).

TLS, like SSL, works using digital certificates and a type of cryptography called public key encryption. TLS relies on trusted third-party organizations called certifica authorities (CA). CAs issue digital certificates to service operators that attest the ownership of the operator's key. The certificate contains information about the server name, the owner's identity, a copy of the public ket, and a cryptographic signature of the CA. Certificates can be issud for single or multiple domain names.
Client applications need to make sure that they are communicating with the correct server by verifying the certificate prior to exchanging data. This is done automatically during the handshake by the application, through comparing the signature on the certificate agains a list of trusted CAs.
