Epoxides

The file /etc/network/interfaces available in Debian and its derived distributions allows to define static and dynamic IP addresses for the interfaces, setup routing information and default gateways, masquerading network bonding and more. The default interfaces file looks like the following:

# This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).

source /etc/network/interfaces.d/*

# The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
allow-hotplug usb0
iface usb0 inet dhcp

Where auto starts the interface at boot and iface calls the network interface (in this case lo, loopback). All lines beginning with "auto" specify the interfaces which will be enabled when running "ifup -a", a command executed at boot.

Lines beginning with “iface” have the following syntax:

iface  <interface> <address_family>  <method>

For example:

iface enp2s0 inet dhcp

The following example shows how to setup a network card using DHCP and wpa supplicant:

# cat /etc/network/interfaces
# This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).

source /etc/network/interfaces.d/*

# The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
allow-hotplug usb0
iface usb0 inet dhcp

# The wifi network interface
iface wlxf4f26d13b2bd inet dhcp
	wpa-driver nl80211
	wpa-ssid TP-LINK_2.4GHz_4F7005_2GEXT
	wpa-psk 46781314

  • Where allow-hotplug will start the interface upon event detection.

  • you can set mac address with 'pre-up macchanger -m' inside the iface stanza. But it's better off to use

    macchanger -m 02:0F:B5:23:28:8E wlxf4f26d13b2bd; ifup -v wlxf4f26d13b2bd

When you make a change to a network inteface file, the configuration are not immediately applied. To put the changes into effect, you'll need to restart the network interface.

systemctl restart networking

Then do

sudo ifup -v wlxf4f26d13b2bd

The -v for verbose should produce some output indicating what's happening behind the scenes.

The ifup and ifdown interface scripts are symbolic links to scripts in the /sbin directory. When either of these scripts are called, they require the value of the interface to be specified, such as the above command. These should be the only scripts that the user use to bring up and take down network interfaces.

static IP

To configure a static IP address:

# The wifi network interface
iface wlxf4f26d13b2bd inet static 
	wpa-driver nl80211
	wpa-ssid TP-LINK_2.4GHz_4F7005_2GEXT
	wpa-psk 46781314	
	address 192.168.1.120
	netmask 255.255.255.0
	gateway 192.168.1.1
