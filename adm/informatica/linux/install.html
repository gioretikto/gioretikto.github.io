The install command

install is cp on steroids. It can set file permissions, create directories, take backups, and much more, all from a single command line. It is most often used in makefiles, to copy the compiled program executable and supporting files to the system.

To copy files using the install command use the following syntax:

install [OPTION]… SOURCE… DIRECTORY

This will copy the file at the source to the destination directory with the same name.

Let’s see an example:

install sample.txt test_directory

You can also get a verbose output from the install command with the -v flag.

install -v sample.txt test_directory

How to create directories using the install command?

You can also use the install command for creating directories. Let’s try running the following command in Linux:

install sample.txt new_parent/new_file

The directory ‘new_parent‘ does not exist. Therefore the output we get is:

install: cannot create regular file 'new_parent/new_file': No such file or

However, if we use the -D flag along with the command above then install command creates the parent directory and then performs the copy operation. An example

install -D myexec dest

will either

  1. if dest doesn't already exist, create a file named dest with the content of myexec.

  2. if dest exists as a directory, copy in it myexec.

    3. The command

    install -D -t dest myexec

    ensures that you always end up with the latter case.

    install -D -v sample.txt new_parent/new_file

    in case the directory new_parent does not exist it will be created and sample.txt will be copied in it with the name new_file.
