typedef

A typedef allows the programmer to give a name of his own choice to a data type. For example, another name may be given to any of the standard data types:

typdef int whole_number;

gives the alternative name whole_number to the standard data type int. From then on, the type whole_number may be used to set up memory spaces to hold integer values. So

whole_number num1, num2;

Giving alternative names in this way, is more useful for complex types as a program can be made readable. The typedef feature is quite useful for defining structures, since it avoids the need to repeatedly write struct keyword to refer to a structure. Using it we can refer the structure more concisely.

struct plane_point
{
	int x;
	int y;
};

It can then be given a typede − to avoid repeatedly referring to it as struct point.

typdef struct plane_point point

allow the type point to be used in definitions as an alternative to using struct plane_point. A more concise way is

typedef struct {
	int x;
	int y;
} POINT;


POINT a;
POINT b;

Similarly, the declaration

typedef char *String;

makes String a synonym for char *

typedef does not really create a new type, it merely renames an existing type. Another example is

typedef int *IntPointer;
Intpointer myIntPointer;

The types size_t and off_t, are typdefs defined by the Standard Library.
