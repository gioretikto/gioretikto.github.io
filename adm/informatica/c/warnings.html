Warnings

warning: declaration of ‘name’ shadows a previous local

It is due to the fact that you are declaring a new variable inside your conditionals. To fix, remove the declaration inside the conditionals.

warning: array subscript has type ‘char’ [-Wchar-subscripts]

The line array[c] is very likely a bug, because the type char can be signed or unsigned. The compiler doesn't want you to inadvertantly specify a negative array index. And hence the warning!. It is enough to define the char as unsigned char.

error: function declaration isn’t a prototype [-Werror=strict-prototypes]

void print_menu();

This error is being shown for the function declaration. The compiler is expecting us to pass "void" in case we are not passing any arguments to the function. That is we need to modify the above function to 

void print_menu(void);

error: ‘variable’ undeclared (first use in this function)

You can only declare variables once. The Following code is wrong:

if(1 == 1)
{
	int a = 42;
}

else
{
	int a = 1;
}

printf("a is %d ", a);

The correct declation of a is the following

int a;

if (1 == 1)
{
	a = 42;
}

else
{
	a = 1;
}

printf("a is %d ", a);

error: variable-sized object may not be initialized

int arr_size = 3; int arr[arr_size] = {1, 2, 3}; // error: variable-sized object may not be initialized.

Static objects like arr in this case must be initialized with a constant value and not a variable.

just use

int a[] = {8,2,7,4,5,6,3,1};

Notice that even using the const qualifier for the ARR_SIZE variable:

const int ARR_SIZE = 3; int arr[ARR_SIZE] = {1, 2, 3};

since ARR_SIZE is still a variable, the error is still present.
