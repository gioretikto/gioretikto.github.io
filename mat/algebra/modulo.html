Aritmetica modulare

We focus in this paragraph on an important relation in ℤ, the congruence relation modulo a positive integer n. This relation identifies two integers if and only if their difference is a multiple of n. Questa relazione quindi si può pensare come un'uguaglianza a meno di multipli di n.

Definition 4.0 - Let n be a given natural number. The Congruence modulo relation is defined thus:

a ≡ b (modulo n)   ⇔   a − b = nh   for some h ∈ ℤ

For example, we can write

38 ≡ 14   (mod 12)

because 38 − 14 = 24, which is a multiple of 12.

Proposition 4.1. Every integer a is congruent modulo n to a unique integer r such that 0 ≤ r < n.

Proof. As we may always divide an integer a by n (see proposition 2.3), if we denote by r the remainder of the division every integer a is congruent modulo n to an integer r such that 0 ≤ r < n, in fact a = r + nh ⇒ a − r = nh.□

The set of the integers congruent modulo n (with n > 1) forms a group. Consideriamo ad esempio il gruppo degli interi modulo 6. Questo gruppo consiste nell'insieme di sei elementi,

{0, 1, 2, 3, 4, 5}

e di un'operazione chiamata addizione modulo 6, che può essere descritta nel modo seguente: Si immaginino i numeri compresi tra 0 e 5 distribuiti attorno la circonferenza di un cerchio. Per sommare due numeri h e k, si inizi con h e ci si muova in senso orario di k unità attorno al cerchio: h + k è il punto finale. Ad esmpio, 3 + 3 = 0, 3 + 5 = 2, e così via. L'insieme {0, 1, 2, 3, 4, 5} con questa operazione è chiamato gruppo degli interi modulo 6 ed è rappresentato dal simbolo ℤ6.

Example - Suppose you look at a clock and want to know what time it will be five hours from now. If it is currently 10 o'clock you do not say "15 o' clock" but rather "3 o' clock". What you do when intepreting a clock is to subtract multiples of 12 when necessary. 54 hours form now it will be "6 o' clock". In fact all you are concerned with the remainder after division by 12:

10 + 44 = 54 = 4 * 12 + 6.

we write

44 + 10 ≡ 6 (mod 12)

The next proposition shows that the modular operator ≡ is an equivalence relation.

Proposition 4.2. For any integer n > 0. The congruent modulo n relation (≡n) on ℤ is an equivalence relation.

Proof. To show that congruent modulo n is an equivalent relation, we must prove that ≡n is reflexive, symmetric and transitive.

Reflexive. If a ∈ ℤ, we need to show that a ≡ a. We must show that there is a h ∈ ℤ such that a − a = hn. This is simply k = 0.

Symmetric. Let a, b ∈ ℤ, assume that a ≡ b. We have to show that b ≡ a. If there exists an integer h ∈ ℤ so that a − b = hn then b − a = −hn holds true because (−h) is also an integer, so ≡ is symmetric.

Transitive. Let a, b ∈ ℤ, assume that a ≡ b and b ≡ c, we have to show that a ≡ c. We know from the definition of ≡ that there are integers h1 and h2 such that

a − b = h1n   and   b − c = h2n

Adding these two equations gives a − c = (h1 + h2)n

Since h1 + h2 ∈ ℤ; ≡n is transitive.□

Notice that an integer is congruent to every other integer modulo 1, that is to say congruence modulo 1 is a trivial equivalence relation; for this reason in what follows we shall in general assume n > 1.

Proposition 4.3. Let a, b, c, d ∈ ℤ. For any integer n > 0 the following properties hold

a ≡ b (mod n),   c ≡ d (mod n)

 ⇓

  • a + c ≡ b + d (mod n)

  • ac ≡ bd (mod n)

Proof.

a ≡ b (mod n) ⇔ a − b = hn

c ≡ d (mod n) ⇔ c − d = kn

by adding up the two relations: a + c − (b + d) = (h + k)n cioè a + c ≡ b + d (mod n). Analogously ac − bd = ac − ad + ad − bd = a(c − d) + (a − b) = akn + hnd = (ak + hd)n thus ac ≡ bd (mod n).□

Equivalence classes

Equivalence classese are known as congruence classes modulo n

Given an integer a, we shall denote by [a]n, or, if no confusion may possibly arise, by [a], or by ā, the congruence class of a modulo n, that is,

[a]n = {b ∈ ℤ | b ≡n a} = { a + hn | h ∈ ℤ } = {..., a −2n, a −n, a, a + n, a + 2n, ...}.

i.e.: congruent integers to a modulo n are those that may written in the form a + nh for a particular h ∈ ℤ. Indeed, if b ≡n a thus b − a is a multiple of n, so b − a = nh for a particular h ∈ ℤ, i.e.: b = a + nh. This is the reason why equivalence classes modulo n are also called residue classes modulo n.

In base alla teoria vista nel capitolo precedente, tale relazione ripartisce allora ℤ in classi di equivalenza non vuote e disgiunte: in ogni classe vi sono tutti i numeri interi che hanno lo stesso resto quando li si divide per n. Dal momento che il resto r della divisione per n deve soddisfare 0 ≤ r < n, i possibili resti sono 0, 1, 2, . . . , n − 1, quindi avremo esattamente n classi di equivalenza.

Definition 4.4. An equivalence class modulo n consists of those integers which have the same remainder on division by n.

  • [0] = {integers that, divided by n, give a remainder equal to 0} = {kn | k ∈ ℤ} = {multiples of n};

  • [1] ={integers that, divided by n, give a remainder equal to 1} = { kn + 1 | k ∈ ℤ}

  • [2] = {integers that, divided by n, give a remainder equal to 1 = = {kn + 2 | k ∈ ℤ}

  • [n-1] = {integers that, divided by n, give a remainder equal to n − 1} = = { kn + n − 1 | k ∈ ℤ}.

If n = 2, we only two possible remainder, 0 and 1 and consequently two classes, [0] e [1]: the first contains all integers {2k | k ∈ ℤ} the second all integers 2k + 1 | k ∈ ℤ }, i.e. the partition induced on ℤ is that of even and odd numbers.

For example the class [2]9 is made by integers of the form 2 + 9h al variare di h ∈ ℤ; per h uguale a 0, 1, 2, 3, etc. otteniamo cosı̀ 2 = 2 + 9 · 0, 11 = 2 + 9 · 1, 20 = 2 + 9 · 2, 29 = 2 + 9 · 3, e cosı̀ via, for negative h we obtain −7 = 2 + 9 · (−1), −16 = 2 + 9 · (−2), −25 = 2 + 9 · (−3), and so on. We have thus:

[2]9 = {... , −25, −16, −7, 2, 11, 20, 29, 38, ...};

possiamo visualizzare la classe di resto di 2 modulo 9 come l’insieme costituito dai numeri che incontriamo partendo da 2 ed percorrendo l’intera lista dei numeri interi facendo passi di lunghezza 9, sia nel verso positivo che in quello negativo. Le altri classi sono:

Possiamo visualizzare la classe di resto di 2 modulo 9 come l’insieme costituito dai numeri che incontriamo partendo da 2 ed percorrendo l’intera lista dei numeri interi facendo passi di lunghezza 9, sia nel verso positivo che in quello negativo. Scegliamo un qualsiasi altro numero in [2]9 , ad esempio 20. Qual è la classe di resto di 20 modulo 9? Se ci pensiamo un attimo, ci accorgiamo che è la stessa classe trovata per 2. Infatti ci possiamo spostare da 20 a 2 (con due passi “all’indietro” di lunghezza 9) e da qui, sempre con passi di lunghezza 9, raggiungere tutti i numeri in [2]9 (per esempio, siccome con cinque passi all’indietro, da 2 si raggiunge −43 = 2 + 9(−5), si potrà raggiungere −43 da 20 effettuando sette passi all’indietro; i primi due passi portano a 2, con i rimanenti cinque si arriva a −43). Vediamo cosı̀ che ogni numero in [2]9 è anche in [20]9. Viceversa, ragionando in modo analogo ma invertendo i ruoli di 20 e 2, scopriamo che, partendo da 2 ed muovendoci solo con passi di lunghezza 9, siccome possiamo raggiungere 20 possiamo anche raggiungere ogni numero in [20]9, quindi ogni numero in [20]9 è anche in [2]9. Mettendo insieme questa informazione con la precedente, concludiamo che [20]9 = [2]9. Questa è una regola generale, si ha infatti, per ogni coppia di interi a e b e per ogni intero positivo n:

a ≡n b  ⇐⇒  [a]n = [b]n

The set of these residue classes is the quotient set of ℤ indicated with ℤn, which consists of n distinct residue classes modulo n::

ℤn = {[0]n, [1]n, [2]n, . . . , [n − 1]n}

Thus, using the congruence relation modulo n

we passed from the infinite set ℤ to the finite set ℤn.

This passage from infinite to finite is often very useful, especially when calculations or checks are to be performed with a computer which, as is well known, only works with finitely many objects.

Come tutte le relazioni di equivalenza, la relazione di congruenza è una sorta di “uguaglianza in senso largo”: consideriamo due numeri “uguali” se lo sono “a meno di mutipli di n”. The symbol ≡ has been introduced by Gauss in the Disquisitiones, by virtue of the analogy with the equality relation. Ora, la relazione di equivalenza per eccellenza, l’uguaglianza, verifica l’ovvia proprietà che se a = b e a′ = b′ allora a + a′ = b + b′ e aa′ = bb′: vediamo ora che anche la congruenza modulo n si comporta esattamente nello stesso modo.

Notiamo anche che ciascuna delle classi di resto si può rappresentare in tanti (addirittura infiniti) modi diversi, ad esempio [2]6 = [14]6 = [6000002]6, (6000002 = 2 (mod 6)) quindi anche ℤn si potrà descrivere in molti modi. Ad esempio,

ℤn = {[1]n, [2]n, ..., [n − 1]n , [n]n},

Operations on ℤn

Definition 4.6. Let [a] and [b] be elements of ℤn, then By proposition 4.2 the following operations can be defined

[a]n + [b]n := [a + b]n

[a]n ⋅ [b]n := [a x b]n

for each a, b ∈ ℤ. These operations are well defined, and do not depend on the representative elements chosen.

Cosa significa? La somma tra due classi di resto A e B (modulo n) si ottiene prendendo un numero a in A ed uno b in B, sommando questi due numeri e considerando come risultato la classe a cui appartiene a + b. Infatti, come abbiamo visto sopra, se a ∈ A e b ∈ B allora A= [a]n e B = [b]n, e la classe a cui appartiene a + b è [a + b]n. Ripetiamolo ancora una volta: questa “classe somma” non dipende dalla scelta di quale particolare elemento a abbiamo selezionato in A e quale b in B: se a 0 ∈ [a]n e b 0 ∈ [b]n allora a ≡n a' e b ≡n b', then by Proposition 4.2, a + b ≡n a' + b', cioè [a' + b']n = [a + b]n. Analogo discorso vale per la moltiplicazione.

Vediamo che mentre la tavola per l’addizione è molto simile a quella che abbiamo costruito per ℤ5, quella relativa alla moltiplicazione è molto diversa. Nel caso di ℤ5, esclusa la riga intestata da zero, che contiene solo zeri, in ogni riga sono rappresentati (una ed una sola volta) tutti gli elementi di ℤ5. Questa differenza è molto significativa: ci dice che ℤ5 e ℤ6 sono strutture algebriche molto diverse tra loro. In ℤ5 ogni elemento diverso dallo zero è, come si dice, invertibile, cioè moltiplicato per un opportuno elemento dà [1]5, che è, come accennato sopra, elemento neutro rispetto alla moltiplicazione in ℤ5. Questo si vede dalla tavola di Cayley per la moltiplicazione di ℤ5: in ogni riga, esclusa quella intestata da 0, appare 1. Ad esempio, nella riga intestata da 2 abbiamo trovato 1 in corrispondenza della colonna intestata da 3, perché [2]5 · [3]5 = [1]5; possiamo esprimere questo fatto dicendo, appunto, che [2]5 è invertibile in ℤ5 e che [3]5 è inverso di [2]3. In questo ℤ5 si comporta come l’anello ℚ dei numeri razionali o quello ℝ dei numeri reali (anelli di questo tipo si chiamano campi), in cui tutti gli elementi diversi da zero hanno inverso (il reciproco) o, equivalentemente, in cui è sempre possibile dividere un elemento per un elemento diverso da zero, e non come ℤ, in cui queste proprietà non valgono. Al contrario, ℤ6, ha proprietà più deboli di quelle che valgono in ℤ. Ad esempio, in ℤ (come in ℚ ed in ℝ, ed anche in ℤ5) vale la legge di annullamento del prodotto: se il prodotto tra due numeri è zero, uno dei due numeri deve essere zero. Questo non vale in ℤ6, come si vede anche dalla tabella; ad esempio [2]6· [3]6 = [0]6, benché [2]6 ≠ [0]6 ≠ [3]6. Vediamo quindi che ℤ, ℤ5 e ℤ6 sono strutture algebriche di natura molto diversa tra loro. Senza entrare in dettagli che qui sarabbero fuori luogo, diciamo che la natura dei singoli anelli ℤn è determinata dalle proprietà aritmetiche dei numeri n che li definiscono. Ad esempio, si dimostra che ℤn è un campo se e solo se n è un numero primo.

The integral domain ℤn when n is a prime

Proposition 4.8. For n > 1, ℤn is an integral domain if and only if n is a prime.

Proof. Suppose first that n is a prime. Let [a] ≠ [0] in ℤn, and suppose [a][b] = 0 for some [b] in ℤn. Now [a][b] = 0 implies that [ab] = 0, and therefore, n| ab. However, [a] ≠ 0 means that n ∤ a. Thus n | ab and n ∤ a imply by Euclid's lemma that n|b, that is [b] = 0. We have shown that if [a] ≠ [0], the only way that [a][b] = 0 can be [0] is for [b] to be [0]. Therefore, ℤn has no zero divisors and is an integral domain.

Suppose now that n is not a prime and [a][b] = 0 holds true. Then n has divisors other than ± 1 and ± n, so there are integers a and b such that

n = ab   where 1 < a < n   and   1 < b < n

This means that [a] ≠ [0], [b] ≠ 0, but

[a] [b] = [ab] = [n] = [0]

Therefore, [a] is a zero divisor in ℤn, and ℤn is not an integral domain. Combining the two cases, we see that n is a prime if and only if ℤn is an integral domain.□

However, it would be wrong to assume that every property holding for the equality holds for congruences as well. For instance, in Z we have the following cancellation property:

ac = bc   implies   a = b,

as long as c ≠0. In other words, ℤ is an integral domain. This property does not hold, in general, for congruences, as the following example shows:

2 · 3 = 6 ≡6 12 = 4 · 3,   but 2 ≠6 4.

Nevertheless, the following result holds:

Proposition 4.8. Let n > 1 be an integer. If ac ≡n bc and if c and n are coprime, that is if GCD(c, n) = 1, then a ≡n b.

Proof. If ac ≡n bc, then n | (a − b)c. As GCD(c, n) = 1, there exist s, t such that 1 = sc + tn and by multiplying both members by a − b, it is possible to write

a − b = (a − b) sc + (a − b) tn = (a − b) c ⋅ s + (a − b)tn

thus n | a − b, because it divides the second member, consequenlty a ≡n b.□

Proposition 4.9. Let n > 1 be an integer. If ac ≡ bc (mod n), then a ≡ b (mod n/d), where d = GCD(c, n).

Proof. The proposition says that it's possible to simplify a congruence eliminating a term, but the module must be changed accordingly. ac ≡ bc (mod n) ⇐⇒ (a − b) c = kn, from which dividing by d = GCD(c, n), we find that

(a − b) c/d = k(n/d)

(the fraction c/d and n/d are integers numbers). Thus n/d divides the product (a − b) c/d and n/d and c/d are coprime: f = c/d, m = n/d, d = gcd(c,n) so gcd(f,m) = gcd(c,n)/d = 1, and so n/d divides a − b, thus a ≡n/d b. □

For instance, 21 = 3 · 7 ≡9 30 = 3 · 10 and 7 ≢ 10 (mod 9), while 7 ≡ 10 (mod 3).

5 ≢ 8 (mod 9), 5 ≡ 8 (mod 9/3).

Remark 4.10. As a consequence of the above, we may see, for instance, that in ℤ5 the same cancellation property holds as in Z; in other words, ℤ5 is, like ℤ, an integral domain, and consequently, since it is finite, a field (see Exercise A1.48). Indeed, if ā· b̄ = 0 with ā, b̄ ∈ ℤ5 and b̄ ≠ = 0, then ā = 0 by Proposition 4.8, as GCD(b, 5) = 1. So ℤ5 is an integral domain. On the other hand, the same results say nothing about ℤ4 . In fact, in ℤ4 there are zero-divisors: we have 2 ·2 = 0 con 2 ≠ 0.

Proposition 4.10. The following properties hold:

  1. if a ≡n b and d | n, then a ≡d b;

  2. if a ≡n b and a ≡m b, then a ≡ b (mod lcm(n, m)).

Proof. They follow directly from the definition of congruence.□

Exercise 4.0 - On the set ℤ consider the following two equivalence relations:

  1. a ≡ b (mod n), that is a𝓡b if and only if a − b = nh for some integer h.

  2. a𝓢b if and only if if a and b have the same remainder when divided by n.

Show that a ≡ b (mod n) if and only if a𝓢b; and the two equivalence relations determine the same classes.

Solution. We prove first that a ≡ b (mod n), implies a𝓢b. If

a − b = hn   for some integer h   (2.0)

then a and b have the same remainder when divided by n. Suppose that a has been divided by n, producing the quotient q and the remainder r with 0 ≤ r < n. This means that a = nq +r. Replacing a in (2.0) with nq + r yields

b = a − hn = (nq + r) − hn = n(q − h) + r

Let's now divide b by n:

b / n = [n(q − h) + r]/n = (q − h) + r/n

This means that the quotient of the division is the integer q − h, and the remainder is r. Thus, a and b have the same remainder when divided by n.

Let’s now prove the converse: a𝓢b, implies a ≡n b. By hypothesis a and b have the same remainder when divided by n; call this common remainder r. Then a = nh + r and b = nk + r with h and k integers, and 0 ≤ r < n. To prove that a ≡n b we need to check whether a − b is divisible by n. Using the information we have for a and b we obtain

a − b = (nh + r) − (nk + r) = nh − nk = n(h − k)

The number h − k is an integer because h and k are integers. Thus a ≡n b.

Part 2. We must prove that the relations "≡m" and "𝓢" are equal. By definition [a]m = {b ∈ ℤ| b ≡ a (mod m)} and [a]𝓢 = {b ∈ ℤ|b𝓢a}. Let b ∈ [0]m, or equivalently, let a ≡m b. By part 1 this is equivalent to stating that a𝓢b. In turn, this is equivalent to saying that b ∈ [a]𝓢. Since b ∈ [a]m if and only if b ∈ [a]m, the two equivalence classes are identical.□

It is important to know when two relations produce the same equivalence classes, because it is then possible to use either one of the two relations, depending on which one is handier in a given setting.
